Simple reading sequence:
- First confirm the official state in `output/project_roadmap_status.txt`.
- Then confirm the formal pause logic in `output/next_step_after_v3_pause_resolution.*`.
- Then confirm the data-policy boundary in `output/phase_0b_final_data_policy_closeout.*`.
- Then confirm the reopen threshold in `output/layer2_reopen_threshold_governance_note.*`.
- Then use `output/layer2_governance_operator_guide.txt` and `output/layer2_governance_toolkit_index.json` as operational tooling for the review workflow.
- Only after that, read historical artifacts to understand why the current state exists.

Bottom line:
- If you want to know what is officially active now, use the current governance artifacts first.
- If you want to know why the project arrived here, use the historical artifacts second.
- If an artifact suggests a new experiment but is older than the documented pause flow, do not treat it as active authority.
