Purpose:
- Menjelaskan artifact governance utama dan ketergantungan sinkronisasi minimumnya.
- Menjawab pertanyaan: jika satu artifact berubah, artifact mana yang minimal harus ikut dicek agar wording dan pointer tidak drift.
- Menjaga paket governance aktif tetap rapi tanpa membingkai sync map ini sebagai source-of-truth baru atau approval artifact.

How to use this sync map:
1. Tentukan artifact yang berubah.
2. Cari artifact itu dalam kelompoknya di bawah.
3. Ikuti minimum sync rules yang relevan.
4. Jika perubahan menyentuh istilah status resmi, lane aktif, limitation aktif, atau urutan baca, cek juga quickstart, registry, dan toolkit index.

High-risk drift patterns to avoid:
- Registry menyebut artifact baru, tetapi how-to-read atau toolkit index belum menunjuk ke artifact itu.
- Quickstart masih menyebut lane aktif lama setelah `post_pause_active_lane_decision.*` berubah.
- Limitation register berubah, tetapi notes index atau boundary note masih memakai wording prioritas/interpretasi lama.
- Source-of-truth bergeser, tetapi discovery/helper artifacts masih merangkum status lama.
- Future reopen tooling berubah, tetapi package closeout dan registry tidak menunjukkan toolkit terbaru.
- Artifact helper dibaca seperti authority baru dan mulai berbeda wording dari source-of-truth resmi.

Forbidden interpretation:
- Sync map ini bukan source-of-truth baru.
- Sync map ini bukan artifact approval.
- Sync map ini bukan izin untuk mengubah governance state.
- Sync map ini bukan izin backtest, retest, paper trading, Phase C, atau coding strategi baru.
