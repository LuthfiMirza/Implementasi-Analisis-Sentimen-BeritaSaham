Layer 2 Governance Operator Guide
=================================

Reference date:
- 2026-04-24

Current official state:
- current_focus_state = strategy_rebuild_layer2_single_variable_lever_tests_paused_after_v3_rejection
- official_state = maintain_framework_governance_gate_blocked_until_new_governance_facing_evidence_changes_threshold_results_on_updated_provisional_core_stack
- formal_resolution = remain_in_documented_pause

Toolkit purpose:
- Memberi alur kerja ringkas dan disiplin untuk reviewer baru yang menilai proposal reopen Layer 2.
- Menjaga agar review tetap konsisten dengan documented pause dan tidak kembali ke hopeful interpretation.

Operational sequence:
1. Baca source-of-truth aktif.
   Mulai dari:
   - `output/project_roadmap_status.txt`
   - `output/next_step_after_v3_pause_resolution.*`
   - `output/phase_0b_final_data_policy_closeout.*`
   - `output/layer2_reopen_threshold_governance_note.*`
2. Cek governance registry dan how-to-read note.
   Baca:
   - `output/governance_artifact_registry.*`
   - `output/how_to_read_current_governance_artifacts.txt`
3. Isi reopen request template.
   Gunakan:
   - `output/layer2_reopen_request_template.*`
4. Nilai dengan intake checklist.
   Gunakan:
   - `output/governance_change_intake_checklist.*`
5. Bandingkan dengan contoh reject/hold.
   Baca:
   - `output/example_layer2_reopen_request_reject_case.txt`
   - `output/example_layer2_reopen_request_hold_case.txt`
   - `output/example_layer2_reopen_review_outcomes.txt`
6. Tulis decision memo.
   Gunakan:
   - `output/layer2_reopen_decision_memo_template.*`
7. Simpan sesuai intake log policy.
   Ikuti:
   - `output/governance_intake_log_policy.txt`

When the outcome must be `reject`:
- Proposal hanya memakai artifact historical atau superseded sebagai dasar utama.
- Proposal adalah disguised reopen dari:
  - `absolute_momentum_floor_on_return_20d`
  - `entry_signal_persistence_minimum_confirmation_days`
  - `stock_trend_quality_gate_via_short_term_ema_slope`
- Proposal hanya variasi threshold, range, persistence wording, atau slope wording dari jalur yang sudah ditolak.
- Failure mechanism yang diklaim tidak benar-benar baru.
- Frozen stack tidak dijaga utuh.
- Proposal diam-diam membuka scope terlarang.

When the outcome must be `hold_in_documented_pause`:
- Proposal cukup disiplin secara format dan tidak jatuh sebagai disguised reopen yang jelas.
- Proposal menjaga Layer 1, Layer 3, Layer 4, Layer 5, walk-forward split, dan realism cost policy tetap unchanged.
- Proposal tidak membuka retest, paper trading, Phase C, coding strategi baru, atau backtest execution.
- Tetapi proposal belum menyertakan artifact governance-facing override yang lebih baru dan valid.

When the outcome can be `eligible_for_formal_review_but_not_auto_execution`:
- Semua pemeriksaan novelty, failure mechanism, frozen stack, dan forbidden scope lolos.
- Ada artifact governance-facing override yang benar-benar lebih baru dan valid.
- Override artifact itu secara eksplisit mengoverride documented pause, bukan hanya memberi ide backlog.

Mandatory caution:
- Tidak satu pun langkah dalam toolkit ini adalah izin otomatis untuk:
  - backtest
  - retest
  - paper trading
  - Phase C
  - coding strategi baru

Bottom line:
- Toolkit ini dipakai untuk review governance yang seragam.
- Selama belum ada override artifact yang lebih baru dan valid, documented pause tetap keputusan resmi aktif.
